LVM による性能劣化 (3rd)

前回の記事は ここ を参照。

今回は SATA SSD を持つ物理マシンを使用。

機種は MacBook Air (13-inch, Mid 2013) である。

OS は Fedora 43 を使用。

LVM 使用による性能への影響

オレンジ (LVM 未使用) を基準として他を見てほしい。

赤 (LVM-thin) の Sequential write が落ち込むのが目立つ。BAD

スナップショットがあると性能は劣化するのか？ (LVM-thick)

青 (スナップショット無) を基準として他を見てほしい。

read に関しては Sequential と Random のどちらもスナップショットによる性能劣化は見られない。

write に関しては Sequential と Random のどちらもスナップショットが一つ存在するだけで激しく性能が劣化し、スナップショット数が多くなる (赤→オレンジ→緑→紫) ほど強く劣化する。BAD

Random write は数値が小さいため棒 (赤、オレンジ、緑、紫) が見えないと思う。必要に応じて拡大してほしい。

スナップショットがあると性能は劣化するのか？ (LVM-thin)

青 (スナップショット無) を基準として他を見てほしい。

(所々妙な数値があるが) LVM-thin はスナップショットによる性能劣化が無い。GOOD

まとめ

LVM-thick はスナップショットがあると write 性能がガツンと落ち、スナップショット数が増えるほど write 性能が落ちる。

LVM-thin はそんなことは無い。
